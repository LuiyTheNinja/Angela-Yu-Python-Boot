Standard development Workflow

Step 1. Define Requirements
What problem does this solve?
Who is the user?
What is the minimum viable product(MVP)?
What features can be added later?

Step 2. Project planning
Requirements
Feature List
Project Scope
Roadmap
Known Limitations
# Create flowcharts or diagrams for major functionality. 
Example:
User
 ↓
CLI Interface
 ↓
Python Functions
 ↓
JSON Storage
 ↓
Reports

Step 3. Repository Structure
project-name/
│
├── README.md
├── LICENSE
├── .gitignore
├── requirements.txt
│
├── docs/
│   ├── requirements.md
│   ├── architecture.md
│   ├── roadmap.md
│   └── changelog.md
│
├── src/
│
├── tests/
│
└── data/

Step 4. git Workflow
branches
 - main
 - development
 - feature/*
Example:
feature-json-storage
feature-export-reports
feature-monitoring
feature-alerting

Step 5. Issue Tracking
Create Github Issues for every feature.

Example: #1 Create CLI

#2 Add JSON storage

#3 Add reporting

#4 Write tests

#5 Create README

Minor = New features
